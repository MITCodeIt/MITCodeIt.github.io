In this codelab, you'll learn how to use the Canvas and Ball components.

What you'll learn

  • Use the Designer View to create the user interface.
  • Use the Blocks View to add functionality to the app.
  • Use the Canvas and Ball components.
  • Use Event Handlers.

Create a new project called BallBounce (no spaces).

The Designer View shows you what the app looks like to the user.

In the Palette section, open the Drawing and Animation drawer and drag a Canvas component onto the viewer.

Challenges

The following challenges will require you to modify the Properties section of the Screen and Canvas components.

  1. Turn off the Scrollable setting of the Screen.
  2. Change the height and width of the canvas to fill parent.

ball-bounce-1

To receive this checkoff, you must

  • Have a canvas that occupies the full screen
  • Describe each of the four panels on the Designer View
  • Explain what it means for a component width or height to "fill parent"

Designer View

Open the Drawing and Animation drawer and drag a Ball component onto the viewer. Adjust the Radius property to make the ball bigger.

Blocks View

Open the Ball drawer and drag the when Ball1.Flung block onto the workspace.

Click on the Ball drawer again and plug in the set Ball1.Speed and set Ball1.Heading blocks into the when Ball1.Flung Event Handler.

Mouse over the speed parameter of the when Ball1.Flung block. Plug the get speed block into the set Ball1.Speed block.

Challenges

  1. Set the Ball's heading to the heading at which the ball was flung.

ball-bounce-2

To receive this checkoff, you must

  • Have a ball that a user can fling across the screen
  • Describe the difference between the Designer and Blocks Views
  • Explain how you used an Event Handler for this part
  • Indicate what color Event Handler blocks are
  • Explain what the get speed and get heading blocks do

Notice that the ball stops moving when it hits the edge of the screen.

To make the ball bounce off the edge of the screen, drag the when Ball1.Edge Reached do event handler into the workspace.

Then add the call Ball1.Bounce block into the event handler.

Challenges

  1. Add the necessary block to make the ball bounce off the edge it reaches.

ball-bounce-3

To receive this checkoff, you must

  • Have a ball that a user can fling across the screen and that bounces off the edge of the screen
  • Explain how you used an Event Handler for this part
  • Explain how you made the ball bounce off the correct edge

You have used App Inventor to create a Ball Bounce game.

What we've covered

  • Use the Blocks View to add functionality to the app.
  • Use the Canvas and Ball components.

 If you finish early, implement one or more of the following features for a bonus checkoff.

ball-bounce-bonus

To receive this checkoff, you must implement one or more of the following

  • Change the color of the ball based on how fast it is moving or which edge it reaches.
  • Make the ball slow down and stop after it is flung.
  • Give the ball obstacles or targets to hit.
  • Introduce a paddle for intercepting the ball, like a Pong game.
  • Make the ball say something based on which edge it reaches.
